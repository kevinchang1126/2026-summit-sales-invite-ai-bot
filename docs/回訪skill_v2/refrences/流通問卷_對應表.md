# 流通問卷對應表

本文件供生成流通業說帖時載入使用。精簡版，保留業務回訪最需要的訊號解讀與話術切入。

## 題組速查

| 題組 | 用途 |
| --- | --- |
| Q1 | 判斷是否願意被接觸與深入交流（高商機錨點）|
| Q4 | 判斷客戶最在意的投資目的與經營議題（主判斷題）|
| Q5 | 判斷導入溫度與推進節奏 |

**注意**：流通版的 Q4（AI 導入關鍵因素）是主判斷題，與製造版 Q4（AI 應用程度）定義不同。

## Q1_活動後協助需求

| 代碼 | 解讀 | 話術切入 |
| --- | --- | --- |
| `Q1_VISIT` ★ | 高商機，願意到府討論 | 直接進需求交流，安排顧問訪談 |
| `Q1_REVIEW_PROCESS` ★ | 高商機，接受流程盤點 | 顧問式流程盤點切入 |
| `Q1_EXPLAIN_SOLUTION` | 中高，對方案有興趣 | 先寄對應場景資料再安排訪談 |
| `Q1_OTHER` | 待判斷 | 需看填寫內容 |

## Q4_AI導入關鍵因素（主判斷題）

依 `流通活動後推進攻略` 的觀念，Q4 是流通場的主判斷題。要先回答：客戶想靠 AI 解決什麼經營問題？

| 代碼 | 解讀 | 話術切入 | 建議場景 |
| --- | --- | --- | --- |
| `Q4_COMPETITION` | 因應市場或同業競爭 | 通路反應速度、多元接單、預測備貨 | 多元接單 Agent、預測備貨 Agent |
| `Q4_REVENUE` | 推動營收成長（高商機主軸）| 增單不增人、通路拓展、接單效率 | 多元接單 Agent、預測備貨 Agent |
| `Q4_EFFICIENCY` | 提升營運效率與員工體驗（高商機主軸）| 減人坑、減重工、跨系統流程效率 | 多元接單 Agent、通路對帳 Agent |
| `Q4_CUSTOMER_EXP` | 提升客戶體驗與服務效率 | 前台接單、客服、會員互動 | 客服 Agent、接單 Agent |
| `Q4_RESILIENCE` | 強化企業韌性 | 備貨、庫存、供應韌性、異常應變 | 預測備貨 Agent、效期觀測 Agent |
| `Q4_SECURITY` | 強化資安與資訊治理能力 | 先談治理與權限，再談場景 | 治理型提案，不推高自主 Agent |
| `Q4_SUSTAINABILITY` | 企業永續與創新 | 中長期轉型、代表性示範場景 | 依客戶方向決定 |
| `Q4_OTHER` | 待判斷 | 需看填寫內容 | — |

**觸發治理語言**：`Q4_SECURITY`

## 典型訊號組合與回訪主軸

### 組合 A：`Q1_VISIT + Q4_REVENUE + Q5_HALF_YEAR`

**判斷**：P1 高機會，願意到府 + 營收成長導向 + 短期內要動

**回訪主軸**：
- 主場景：多元接單 Agent 或預測備貨 Agent
- 主講師：黃昱凱副總裁（數位勞動力）
- 補強：朱浩所長（流通導入節奏）
- 治理語言：弱（因主軸是營收）

### 組合 B：`Q1_REVIEW_PROCESS + Q4_EFFICIENCY + Q5_ONE_YEAR`

**判斷**：P1，接受流程盤點 + 效率主軸 + 中期規劃

**回訪主軸**：
- 主場景：通路對帳 Agent、多元接單 Agent
- 主講師：黃盈彰總經理（智慧底座）
- 補強：朱浩所長（流通 PoC 節奏）
- 治理語言：弱至中

### 組合 C：`Q1_EXPLAIN_SOLUTION + Q4_RESILIENCE + Q5_TWO_YEAR`

**判斷**：P2/P3 邊界（依其他訊號），韌性主軸，中期評估

**回訪主軸**：
- 主場景：預測備貨 Agent、效期觀測 Agent
- 主講師：黃盈彰總經理（供應鏈風險）
- 補強：朱浩所長、洪春暉所長
- 治理語言：弱

### 組合 D：`Q1_EXPLAIN_SOLUTION + Q4_SECURITY + Q5_HALF_YEAR`

**判斷**：P1，治理主軸、短期導入

**回訪主軸**：
- 主軸：先談權限、資料邊界、治理，再談場景
- 主講師：李義訓副總裁、黃盈彰總經理
- 補強：林大馗董事（Shadow AI、可信任 AI）、朱浩所長
- 治理語言：強，必須用核心語言

### 組合 E：`Q4_COMPETITION + Q5_ONE_YEAR`

**判斷**：P2，在意市場競爭，中期推進

**回訪主軸**：
- 主場景：通路反應速度、前台服務效率
- 主講師：李義訓副總裁（企業級 AI 能力）
- 補強：朱浩所長、洪春暉所長
- 治理語言：弱

## 高商機訊號清單（供分類用）

符合任一條件，歸 P1：

- `Q1_VISIT`
- `Q1_REVIEW_PROCESS`
- `Q1_EXPLAIN_SOLUTION` **且**同時有 `Q5_HALF_YEAR` / `Q5_ONE_YEAR` / `Q5_ADOPTED`
- `Q4_REVENUE` 或 `Q4_EFFICIENCY`，**且**同時有 `Q5_HALF_YEAR` / `Q5_ONE_YEAR` / `Q5_ADOPTED`

## 流通版推進優先級

| 優先級 | 條件 | 回訪策略 |
| --- | --- | --- |
| L1 立即推進 | Q1 願意到府或流程盤點 + Q5 為半年內/1 年內/已導入 | 直接安排訪談，聚焦單一場景立案 |
| L2 積極推進 | Q1 想了解方案，Q4 動機明確、Q5 有時程 | 先寄對應資料，1 週內安排短訪談 |
| L3 案例培育 | Q4 動機明確，但 Q5 為二年內或尚未評估 | 用案例、攻略與活動升溫 |
| L4 長期培育 | Q1 不明確、Q5 尚未評估、動機模糊 | 內容經營為主，不強推 |

對應到 skill 的 tier：L1 → P1、L2 → P2、L3 → P3、L4 → P4。

## 流通版回訪注意事項

1. **Q4 是流通主判斷題**，不可跳過直接談場景
2. 流通業同時有多通路、多據點、多系統，講治理時要強調「資料分散」的現實
3. 朱浩所長的語言風格（可控、高 ROI、分級導入、先單店 PoC）很適合混用，但不能讓朱浩所長取代鼎新主敘事
4. 若客戶主軸是營收成長（Q4_REVENUE），優先用「增單不增人」「前台數智分身」這類直接對應營收的語言
5. 若客戶訊號含 `Q4_SECURITY`，要先降溫技術討論，從資料權限、個資、治理先談起
6. 流通業前台（客服、接單、會員）是 AI 很容易切入的低阻力場景，適合作為第一個試點建議
