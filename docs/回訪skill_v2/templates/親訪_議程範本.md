# 親訪議程範本

## 適用情境

業務親自到客戶公司進行回訪。特性：面對面、時間較長、可帶資料、可雙向討論。

## 時長與結構規範

| tier | 建議時長 | 結構層級 |
| --- | --- | --- |
| P1 | 45-60 分鐘 | 完整議程 + 場景盤點 |
| P2 | 30-45 分鐘 | 縮減議程 + 探索 |
| P3 | 30 分鐘 | 輕量互動 + 分享 |
| P4 | 不建議親訪 | 除非客戶主動邀請 |

**原則**：P4 客戶不主動推親訪。

---

## 親訪專屬禁令

- ❌ 不可缺少「會談建議流程」區塊
- ❌ 不可缺少「會前準備資料」提醒
- ❌ 議程總時長不可超過 60 分鐘（超過會讓客戶壓力太大）
- ❌ 不可把親訪變成單向簡報（必須有雙向探詢環節）
- ❌ 不可在第一次親訪就拿出合約、報價單
- ❌ 不可同時約 3 位以上鼎新人員出席（會讓客戶有被包圍感）

---

## 必備結構

CONTENT 區塊必須包含以下三段：

1. **會談開場話術**（約 150 字，見面時說）
2. **會談議程**（含時間分配）
3. **會前準備資料清單**（給業務自己看的）

---

## 會談開場話術庫

### P1 開場

> 「{稱呼}您好，很感謝您今天撥時間。這次會議主要想針對年會後您提到的 AI 推進想法，花大概 45-60 分鐘一起做一個場景對焦。我們這邊會帶幾個跟貴公司比較接近的實際案例，中間也會有一些問題想跟您確認，這樣後續如果要推進，方向會比較清楚。」

### P2 開場

> 「{稱呼}您好，很感謝您今天撥時間。今天不會給您壓力，主要是針對您在年會問卷裡的方向，花 30-45 分鐘一起看看哪一塊最貼近貴公司目前的狀況。後續要不要再往下走，看完再決定就好。」

### P3 開場

> 「{稱呼}您好，很感謝您撥時間見面。今天主要想跟您分享幾個同產業企業現在在做的方向，大概 30 分鐘左右。如果有哪一塊讓您覺得跟貴公司有連結，我們再慢慢聊。」

## 議程範本

### P1 議程範本（45-60 分鐘）

```
【會談議程】

00:00-05:00  開場 + 雙方介紹
  - 業務側：自我介紹、此次來訪目的
  - 客戶側：目前關注的 AI 議題快速分享

05:00-15:00  問卷訊號確認
  - 確認客戶勾選的訊號是否仍代表當下狀況
  - 了解背景脈絡（誰在推動、期待成效）
  - 重點：聽，不是說

15:00-30:00  場景盤點
  - 依訊號主場景（如供應鏈 / 財務核銷 / 決策支援），帶 1-2 個同產業案例
  - 引用鼎新講師觀點強化落地邏輯
  - 若含治理訊號，同步談身份 / 權限 / 責任

30:00-45:00  可推進方向對焦
  - 哪些場景最適合先做
  - 資料與系統條件盤點
  - 若 P1 含 Q7_ROI，同步談可量化指標

45:00-55:00  下一步共識
  - 提出 2-3 個可能的下一步選項
  - 由客戶選擇節奏
  - 約定後續時間點

55:00-60:00  收尾 + 資料交付
```

### P2 議程範本（30-45 分鐘）

```
【會談議程】

00:00-05:00  開場 + 感謝
05:00-15:00  客戶現況輕度探詢
15:00-30:00  分享 1-2 個同產業案例
30:00-40:00  探詢有興趣方向
40:00-45:00  約定下一步（寄資料 / 短訪談二擇一）
```

### P3 議程範本（30 分鐘）

```
【會談議程】

00:00-05:00  開場 + 寒暄
05:00-20:00  分享年會重點與同產業趨勢
20:00-25:00  輕度探詢客戶想法
25:00-30:00  約定後續觸點（寄資料 / 邀活動）
```

## 會前準備資料清單

### P1 必備資料

- 客戶勾選的問卷訊號摘要（一頁紙）
- 2 份同產業案例（對應主訊號場景）
- 鼎新智慧底座 / 企業運行空間說明資料
- 若含治理訊號：代理治理一頁紙
- 若含 Q7_ROI：ROI 衡量框架一頁紙

### P2 必備資料

- 客戶勾選的問卷訊號摘要
- 1 份同產業案例
- 年會精華重點摘要

### P3 必備資料

- 年會精華重點摘要
- 1 份與客戶最可能相關產業的入門案例

### 範本 A：製造版 P1，訊號 `Q1_ARRANGE + Q5_SUPPLY_CHAIN + Q7_DATA`

```
【會談開場話術】

【會談議程】

00:00-05:00  開場 + 雙方介紹
05:00-15:00  問卷訊號確認
  - 確認供應鏈議題是否仍為當下焦點
  - 了解誰在推動、期待什麼成效

5. 製造業 ROI 衡量框架參考
```

## 輸出格式對應

親訪內容填入 `## CONTENT` 區塊。格式如下：

```markdown
## CONTENT
【會談開場話術】

（開場話術本文）

【會談議程】

（議程表）

【會前準備資料】

（資料清單）
```

**注意**：
- `【】` 是內層小標，允許使用（不違反「CONTENT 不得用 `##` 子標題」的規則）
- 三段順序固定，不得顛倒
- 議程時間以 `HH:MM` 格式呈現
