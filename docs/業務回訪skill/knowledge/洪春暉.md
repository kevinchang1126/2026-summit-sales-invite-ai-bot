# 洪春暉演講知識庫

## 內容定位

洪春暉的演講主軸是「Agentic AI 重構企業競爭力與轉型新賽局」。這份內容適合支援業務在回訪時，協助客戶從「AI 是否要做」推進到「AI 要落在哪些流程、如何形成可治理的人機協作系統、第一個場景要怎麼選」。

核心用途：

- 判斷客戶是否仍停留在工具試用，或已開始思考流程、資料、組織與治理。
- 對應製造業客戶在供應鏈、採購、資料整合、知識工作、員工接受度上的 AI 導入問題。
- 將問卷 Q1 的協助需求與 Q4-Q8 的成熟度、急迫場景、成效期待、挑戰、投入意願串成回訪話術。

## 問卷對應知識卡

### K-HCH-01：AI 導入已從效率工具變成競爭力議題

對應題項：

- `Q1_協助需求`：想了解 AI 最新應用、對 AI 方案感興趣、安排人員了解需求。
- `Q4_AI應用程度`：尚未起步、局部嘗試、點狀應用。
- `Q8_投入意願`：觀望、積極評估。

知識摘要：

講義與逐字稿都強調，AI 已不是要不要導入，而是如何導入、導入在哪裡、如何做好。企業導入 AI 的效益也不再只看降低成本與提升效率，而是延伸到創新、產品開發、員工滿意、客戶滿意與差異化競爭。

業務回訪切入：

「如果貴公司目前還在觀望，我們可以先不急著談工具，而是先釐清 AI 要幫公司創造哪一種競爭力：提高效率、加速產品或服務創新、提升客戶體驗，還是建立差異化。」

建議追問：

- 目前公司談 AI，比較偏向省成本，還是希望創造新價值？
- 管理層最希望 AI 改善哪一個經營指標？
- 目前是否已有部門自行試用 AI，但還沒有形成正式流程？

### K-HCH-02：Agentic AI 是新型數位勞動力，不只是問答工具

對應題項：

- `Q1_協助需求`：對 AI 方案感興趣、安排人員了解需求。
- `Q4_AI應用程度`：局部嘗試、點狀應用、系統整合。
- `Q6_Agent期待成效`：高頻度、高知識、高經驗。

知識摘要：

講義區分傳統 AI 與 Agentic AI：傳統 AI 偏向一次性回答，Agentic AI 則能規劃任務、使用工具、分階段執行、自我修正。講義也指出 Agent 落實重點包含記憶力、規劃能力、工具使用與產出要求。

業務回訪切入：

「如果現在只是把 AI 當成問答工具，價值會停在個人生產力。下一步可以把 Agent 看成數位勞動力，讓它具備任務記憶、步驟規劃、工具串接與可檢核的產出。」

建議追問：

- 現在 AI 主要用在個人查詢，還是已經進入部門流程？
- 哪些任務需要 Agent 不只回答，而是能追蹤、整理、執行或回報？
- 若要做數位員工，最重要的是記憶、規劃、工具使用，還是產出格式？

### K-HCH-03：供應鏈與採購是 Agentic AI 的高價值快思場景

對應題項：

- `Q5_急迫領域`：生產與供應鏈管理。
- `Q6_Agent期待成效`：高頻度、高知識、高經驗、負荷率。
- `Q8_投入意願`：積極評估、已有明確預算並開始執行。

知識摘要：

講義以供應鏈管理作為「快思」案例：Agentic AI 可協助自動生成採購流程、自動生成採購單，並協助公司設計採購流程與議價。逐字稿也補充，外部環境與關稅變化快速，供應鏈管理需要更即時的資訊整理與決策支援。

業務回訪切入：

「您如果在問卷選到生產與供應鏈，洪春暉老師的案例很適合：供應鏈變動快，AI Agent 可以先從採購流程、供應商比較、採購單生成、交期與風險資訊整理切入。」

建議追問：

- 目前供應鏈最痛的是價格、交期、關稅、缺料，還是供應商風險？
- 採購流程中哪些判斷高度依賴資深人員經驗？
- 是否已有 ERP、採購、供應商資料可供 Agent 串接或查詢？

### K-HCH-04：資料整合與可追溯流程是導入 Agent 的前提

對應題項：

- `Q4_AI應用程度`：點狀應用、系統整合。
- `Q7_AI轉型挑戰`：數據品質不佳或尚未整合、缺乏 AI 應用人才。
- `Q8_投入意願`：積極評估、已有明確預算並開始執行。

知識摘要：

講義提到 AI 可將使用者需求轉為可執行程式碼，建構可解釋、可追溯的資料收集流程。這提醒企業：Agent 導入不是只接一個模型，而是要把資料來源、流程邏輯、追溯機制與輸出檢核一起設計。

業務回訪切入：

「如果貴公司卡在資料分散或品質不穩，第一步不一定是做大型平台，而是先選一個任務，把資料來源、查詢流程、產出格式和覆核方式做成可追溯的小流程。」

建議追問：

- 目前資料分散在哪些系統或部門？
- 有沒有某些報表或查詢，每次都靠人工跨系統整理？
- 若 Agent 產出錯誤，目前公司希望由誰覆核？

### K-HCH-05：Agent Ops 與治理要和導入同步設計

對應題項：

- `Q4_AI應用程度`：系統整合、全面賦能。
- `Q7_AI轉型挑戰`：數據品質不佳或尚未整合、成本過高且 ROI 不明確。
- `Q8_投入意願`：已有明確預算並開始執行。

知識摘要：

講義將「應用領域選擇」與「Agent 管理」列為管理重點，包含建置、評估與測試、人類回饋、監控與追蹤。逐字稿也提到 Agent Ops 與 Agent Security Ops，代表企業需要管理 Agent 的部署、權限、安全、成本、互動與任務執行過程。

業務回訪切入：

「如果貴公司已經準備讓 Agent 進流程，下一步不是只問能做什麼，而是要問怎麼管理。任務範圍、權限、安全、監控、異常處理與人工覆核，都需要在導入時一起設計。」

建議追問：

- 公司是否已定義 Agent 可使用哪些資料與工具？
- Agent 的回覆或行動，是否需要人工審核門檻？
- 是否需要追蹤 Agent 的成本、任務成功率、錯誤率與使用紀錄？

### K-HCH-06：用人機合作矩陣選第一個場景

對應題項：

- `Q6_Agent期待成效`：高頻度、高知識、高經驗、負荷率。
- `Q7_AI轉型挑戰`：員工排斥或恐懼、缺乏 AI 應用人才。
- `Q8_投入意願`：觀望、積極評估。

知識摘要：

洪春暉提出，可用「AI 對人的賦能程度」與「員工想不想跟 AI 合作」來選擇任務。高賦能且員工意願高，是適合作為第一個試點的低垂果實；高賦能但員工抗拒，則需要先處理溝通、教育與角色再設計。

業務回訪切入：

「如果擔心員工抗拒，我們可以先選員工也想被幫忙的痛點，不一定一開始就碰最敏感的工作。第一個場景最好同時具備高價值和高接受度。」

建議追問：

- 哪些工作同仁最希望 AI 幫忙？
- 哪些工作雖然價值高，但員工可能擔心被取代？
- 是否需要先做內部教育、工作坊或小規模試點，降低抗拒？

### K-HCH-07：AI 轉型會推動組織與角色再設計

對應題項：

- `Q1_協助需求`：安排人員了解需求、線下實體體驗活動、線上活動報名。
- `Q4_AI應用程度`：系統整合、全面賦能。
- `Q7_AI轉型挑戰`：員工排斥或恐懼、缺乏 AI 應用人才。

知識摘要：

講義指出，未來會有越來越多 Agent 導入組織，企業結構會因場景軟體化、人機協作與角色調整而改變。結論也強調，在與 AI 共舞下，組織需持續平衡人員、科技、任務與組織架構。

業務回訪切入：

「如果 AI 已經不只是部門工具，而要進入核心流程，我們建議同步討論角色分工：哪些由 AI 做、哪些由人覆核、哪些工作需要重新訓練或重新設計。」

建議追問：

- 導入 AI 後，哪些角色的工作內容會最先改變？
- 公司是否已規劃 AI 使用規範、內訓或角色升級方向？
- 是否需要透過工作坊把管理層、使用者與 IT 的期待先對齊？

### K-HCH-F01：流通業現在要問的是怎麼做 AI，而不是要不要做

對應題項：

- `Q1_活動後協助需求`：到府討論需求、進一步說明解決方案。
- `Q4_AI導入關鍵因素`：推動營收成長、提升客戶體驗與服務效率、提升營運效率與員工體驗。
- `Q5_AI採用時程`：預計半年內導入、預計一年內導入、已導入。

知識摘要：

洪春暉強調，AI 導入已不是「要不要」的問題，而是「如何導入、導入在哪裡、如何做好」。對流通業來說，若現在想透過 AI 推動營收、服務或效率，就不該停留在泛泛了解趨勢，而要開始選擇第一個有價值的經營場景。

業務回訪切入：

「如果現在評估 AI 的目的是營收成長、服務效率或營運效率，我們比較建議先不要從大平台開始談，而是先找一個與經營目標最直接相關的場景，先做出成果。」

建議追問：

- 目前最優先想改善的是營收、服務速度，還是內部效率？
- 公司現在看 AI 比較像競爭工具，還是新的營運能力？
- 如果先選一個場景，最希望在哪一個指標上先看見改善？

### K-HCH-F02：流通業可以把 Agentic AI 視為新型數位勞動力

對應題項：

- `Q4_AI導入關鍵因素`：提升營運效率與員工體驗、提升客戶體驗與服務效率。
- `Q5_AI採用時程`：預計一年內導入、預計二年內導入、已導入。

知識摘要：

洪春暉從全球導入現況指出，Agentic AI 將成為企業新型數位勞動力。對流通業來說，這很適合用來理解接單、客服、跨通路資訊整理、營運工作流程等高頻任務，不再只是個人用的問答工具。

業務回訪切入：

「如果現在的痛點是高頻又費力的營運工作，Agentic AI 比較像數位勞動力，而不是單純聊天工具。我們可以先挑一個高頻流程，看看能不能讓 Agent 先接手部分工作。」

建議追問：

- 哪些工作現在最花時間、最重複、最容易讓同仁疲於奔命？
- 前台與後台之間，哪一段流程最適合先讓 Agent 協助？
- 您期待的是降低人力負荷，還是提升服務速度？

### K-HCH-F03：流通供應與韌性問題，先從資料整合與風險支援開始

對應題項：

- `Q4_AI導入關鍵因素`：強化企業韌性、提升營運效率與員工體驗。
- `Q5_AI採用時程`：預計半年內導入、預計一年內導入、已導入。

知識摘要：

洪春暉用供應鏈變動、關稅與外部環境變化說明 Agentic AI 的價值。若對應到流通場景，可延伸成補貨、缺貨、壓貨、交期、供應波動與多通路協調。這一類問題通常不會先卡在模型，而是先卡在資料分散、資訊更新慢與決策反應不足。

業務回訪切入：

「如果現在更在意韌性與異常反應速度，我們會建議先看資料能不能被整合成可追蹤、可分析的流程，再讓 Agent 協助預警、整理與決策支援。」

建議追問：

- 最近供應、補貨或交期波動最大的壓力來自哪裡？
- 哪些數據現在分散在不同系統或不同通路？
- 若先做一個韌性場景，您希望先解的是缺貨、壓貨，還是異常反應慢？

### K-HCH-F04：流通客戶若重視治理，就先談 Agent Ops 與可追溯

對應題項：

- `Q1_活動後協助需求`：檢視流程狀況、進一步說明解決方案。
- `Q4_AI導入關鍵因素`：強化資安與資訊治理能力。
- `Q5_AI採用時程`：預計半年內導入、預計一年內導入、已導入。

知識摘要：

洪春暉提到，當 AI 進入企業運作，Agent Ops 與 Agent Security Ops 會變成必要配套。對流通業來說，若要讓 Agent 進到跨通路、接單、會員、營運或供應流程，就必須先處理資料權限、流程可追溯、監控、例外處理與人類確認節點。

業務回訪切入：

「如果目前優先考量是資安與治理，我們不建議先談很大的自主 Agent，而是先談可追溯、可監控、可控權限的導入方式，讓 AI 能進流程但不失控。」

建議追問：

- 現在最擔心 AI 帶來的是資料外洩、誤判，還是責任不清？
- 哪些流程若交給 Agent，需要保留人工確認？
- 是否已有權限控管、操作紀錄或稽核要求需要一起考量？

### K-HCH-F05：流通場第一個試點要選高賦能、高合作意願的場景

對應題項：

- `Q1_活動後協助需求`：到府討論需求、檢視流程狀況。
- `Q4_AI導入關鍵因素`：提升營運效率與員工體驗、提升客戶體驗與服務效率。
- `Q5_AI採用時程`：尚未評估、預計二年內導入、預計一年內導入。

知識摘要：

洪春暉提出的人機合作矩陣很適合流通場的第一步選題。若客戶還在評估，不必一開始就碰最敏感的決策流程，而是先選高賦能且同仁願意合作的場景，例如高頻查詢、資料整理、對內協作、服務輔助等。

業務回訪切入：

「如果目前還在評估期，我們通常會先從同仁也想被 AI 幫忙的場景開始，不一定一開始就碰最敏感的決策工作，這樣比較容易做出第一個成功案例。」

建議追問：

- 哪些工作同仁其實最希望有人幫忙分擔？
- 哪些工作價值高，但現在碰了可能引發抗拒？
- 如果先做試點，您傾向先選前台服務、後台營運，還是跨部門協作？

## 依問卷訊號的回訪優先級

### 高機會推進

符合條件：

- `Q1_協助需求` 選「安排人員了解需求，進行 AI 說明與規劃」。
- `Q8_投入意願` 選「已有明確預算並開始執行」或「積極評估」。
- `Q5_急迫領域` 選「生產與供應鏈管理」或已有明確部門痛點。

建議動作：

- 直接安排需求訪談，聚焦一個高價值流程。
- 用 K-HCH-03 或 K-HCH-05 進入供應鏈/採購或治理討論。
- 產出「場景、資料來源、Agent 任務、人工覆核、成效指標」五欄式盤點。

### 可培育推進

符合條件：

- `Q1_協助需求` 選「對 AI 方案感興趣，提供相關資料讓我參考」。
- `Q8_投入意願` 選「觀望」或「積極評估」。
- `Q4_AI應用程度` 為「尚未起步」或「局部嘗試」。

建議動作：

- 先提供 Agentic AI 與傳統 AI 差異、供應鏈案例、企業工作流程案例。
- 邀請參加線上活動或線下體驗。
- 用 K-HCH-01 和 K-HCH-02 協助建立「AI 從工具到數位勞動力」的認知。

### 需先解風險

符合條件：

- `Q7_AI轉型挑戰` 選「數據品質不佳或尚未整合」。
- `Q7_AI轉型挑戰` 選「員工排斥或恐懼」。
- `Q7_AI轉型挑戰` 選「成本過高且 ROI 不明確」。

建議動作：

- 資料問題先用 K-HCH-04 切入小型可追溯流程。
- 員工抗拒先用 K-HCH-06、K-HCH-07 切入人機合作矩陣與教育。
- ROI 不明確先用 K-HCH-01 協助定義不只省成本，也包含創新、滿意度與差異化。

## 業務可用短句

- 「AI 現在不是要不要導入，而是導入在哪裡、怎麼做好。」
- 「Agentic AI 不只是回答問題，而是能規劃、使用工具、追蹤任務並自我修正。」
- 「供應鏈與採購很適合作為 Agentic AI 的第一波高價值場景。」
- 「Agent 上線後，需要 Agent Ops 與安全治理，不只是功能導入。」
- 「第一個場景最好選高賦能、員工也願意合作的工作。」
- 「真正的競爭力，是把資料、流程、決策與執行整合成可治理、可規模化的人機協作系統。」
