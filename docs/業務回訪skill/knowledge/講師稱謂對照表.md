# 講師稱謂對照表

本表提供業務在電話、親訪、LINE、EMAIL 回訪時，引用年會講師觀點的建議稱呼。

使用原則：

- 第一次提及時，優先使用「正式職稱 + 姓名」或「姓名 + 建議對外稱呼」。
- 後續提及可簡化為「某某副總裁 / 某某院長 / 某某所長 / 某某董事 / 某某總經理」。
- 若正式職稱為 `副總裁`，對外稱呼一律使用 `副總裁`，不要簡化成 `副總`。
- 若正式職稱未確認，先不要硬寫職稱，優先使用較安全的「老師」或「講師」。
- 不建議在對客訊息中只直呼姓名，除非業務與客戶已非常熟。

## 各聯繫方式建議

### 電話

- 優先用簡潔稱呼，例如：`李義訓副總裁`、`林大馗董事`、`朱浩所長`。
- 不要連續堆疊全職稱，避免口語太重。

示例：

- `李義訓副總裁在會中有提到，很多企業真正卡住的不是缺工本身，而是經驗沒辦法被留下來。`
- `朱浩所長這場其實很強調，流通業現在更重要的是怎麼把 AI 變成經營能力。`

### 親訪

- 第一次可用較完整稱呼，後續改用簡稱。
- 若客戶層級較高，可優先用正式職稱版本。

示例：

- `KPMG Advisory 執行董事林大馗在會中提到，AI 一旦進入正式營運，治理與風險框架就要先到位。`
- `後面如果用林大馗董事的角度來看，這題會更像是 ROI 與治理並行的問題。`

### LINE

- 以自然、好讀為主，建議用 `某某副總裁 / 某某院長 / 某某所長 / 某某總經理`。
- 避免使用過長正式頭銜。

示例：

- `朱浩所長在流通場有提到，現在比較務實的做法，是先找一個高 ROI 場景做 PoC。`

### EMAIL

- 第一次提及建議使用正式職稱。
- 第二次之後可改為簡稱。

示例：

- `商業發展研究院商業發展與策略研究所資深研究員兼所長朱浩於會中提到，流通業導入 AI 應優先從高價值場景與單點 PoC 開始。`
- `朱浩所長的這個觀點，特別適合用來對應貴司目前的導入節奏。`
